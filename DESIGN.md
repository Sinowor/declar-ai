# DeclarAI Design System

基于 Cargo Wise 设计指南，面向报关自动化制单的桌面端 Electron 应用。

## 设计定位

核心气质：**现代物流 SaaS、轻盈、可信、智能、专业**。

视觉关键词：
- 大留白：报关单复杂度高，界面必须给足呼吸感
- 柔和渐变：紫色+蓝色低饱和渐变表达 AI 智能处理和数据流动
- 圆角卡片：卡片承载报关单、货物明细、AI审核面板
- 清晰数据：表格、编号、件数、重量、状态必须优先可读
- 克制品牌：紫色只用于关键 CTA、拖拽高亮、AI 状态指示

## 组件规范

### 按钮层级

- **Primary**（紫底白字）：AI 提取、提交、保存
- **Secondary**（白底灰边）：取消、返回
- **Ghost**（无边框）：表格行内编辑、删除明细行
- **Danger**（红底白字）：删除申报单

每个页面只有一个 Primary 按钮。多操作用下拉菜单。

按钮文案动词开头：`提取数据` `保存草稿` `添加货物` `AI审核`

### 状态徽章

| 状态 | 颜色 | 图标 |
| --- | --- | --- |
| 草稿 | Muted 灰 | 📄 |
| AI提取中 | Info 蓝 + 脉冲动画 | 🤖 |
| 待人工确认 | Warning 黄 | ⚠️ |
| 已完成 | Success 绿 | ✓ |
| 有错误 | Danger 红 | ✗ |

### 拖拽导入区

- 虚线边框，hover 时变为紫色实线 + 浅紫背景
- 默认显示："拖拽箱单、发票、合同、运单等到此处" + 文件图标
- 支持点击选择文件
- 已导入文件显示为标签列表，每个标签可删除
- 底部显示"开始 AI 提取"按钮

### 货物明细表格

- 首列：序号
- 关键列：货物名称、提单号、集装箱号、件数、重量
- 金额右对齐
- 行内编辑：点击单元格直接修改
- 行操作：删除（>2行时显示）
- 底部汇总行：合计件数、合计重量（紫色加粗）
- 支持添加行

### AI 审核面板

- 右侧滑出面板或底部折叠面板
- 显示 AI 发现的问题列表
- 每条问题：字段名 + 问题描述 + 用户输入框 + 确认按钮
- 已回复的问题显示绿色已处理标记
- 空状态："AI 未发现明显问题，数据质量良好"

### 空状态

没有申报单时：
> 暂无申报单。点击「新建申报单」开始制作您的第一票转关运输货物申报单。

没有导入文件时：
> 拖拽箱单、发票、合同、运单等文件到此处，或点击选择文件。支持 PDF、Excel、Word 格式，也支持 ZIP/RAR 压缩包。

## 交互规范

### 进入/退出锁定模式

- 从列表点击进入某票 → 左侧列表变灰，显示锁图标
- 工作区显示该票详情
- 点击左侧"退出"或 ESC → 返回列表，解锁

### AI 提取流程

1. 用户上传文件 → 文件列表显示
2. 点击「开始 AI 提取」
3. 文件依次显示处理状态（队列中 → 解析中 → 提取中 → 完成）
4. 全部完成后，表单自动填充
5. 低置信度字段黄色高亮，提示用户确认
6. 用户可点击「AI 审核」查看问题列表

### 表单保存

- 手动保存：Ctrl/Cmd + S 或点击保存按钮
- 自动保存：离开页面或关闭时提示保存
- 保存成功 Toast 提示

## 落地检查清单

- 页面是否有清晰标题和主要操作？
- 左侧列表是否支持搜索和折叠？
- 锁定模式下是否阻止切换申报单？
- 拖拽区域是否有明确的视觉反馈？
- 货物明细是否自动汇总？
- 低置信度字段是否有视觉提示？
- 空状态是否给出明确指引？
- 表格金额和数字是否右对齐？
