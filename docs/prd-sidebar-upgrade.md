# PRD: 侧边栏 UI 升级

> 参考 WorkBuddy macOS 风格侧边栏设计，优化信息层级和视觉体验

## Problem Statement

当前侧边栏布局平淡：新建按钮在底部（不显眼）、列表项信息密度低（缺少类型描述）、空状态简陋（纯文字"暂无申报单"）、缺少用户上下文信息。整体视觉缺乏层次感和专业感。

## Solution

参照 macOS 原生应用侧边栏的 4 区域布局模式，重构 DeclarAI 侧边栏：

1. **顶部 CTA**：新建申报单按钮从底部移至顶部，白色圆角卡片 + 阴影 + 加号图标，强制视觉引导
2. **列表信息增强**：每项展示类型 + 状态文字副标题（如"进口报关 · 待确认"）
3. **底部信息栏**：显示当前选中申报单概要，或快捷操作入口
4. **空状态升级**：图标 + 引导标题 + 操作说明，引导用户新建
5. **整体视觉**：间距收紧、hover 态精致、深色模式适配

## User Stories

1. 作为制单员，我希望新建按钮在侧边栏顶部显眼位置，以便进入应用后第一眼就能看到并快速操作。
2. 作为制单员，我希望列表每项展示申报类型和状态文字，以便在不点击的情况下快速了解每份申报单的概况。
3. 作为制单员，我希望侧边栏底部显示当前工作上下文（如选中申报单名称），以便始终知道自己正在处理哪份单证。
4. 作为制单员，我希望空状态有引导性的图标和文字，以便知道如何开始工作。
5. 作为制单员，我希望列表的 hover 和选中态更加精致（微妙圆角、平滑过渡），以便获得更专业的操作体验。
6. 作为制单员，我希望侧边栏在深色模式下依然层次分明，以便在不同光照环境下舒适使用。
7. 作为制单员，我希望搜索框在顶部 CTA 下方、列表上方，与列表形成明确的"过滤/浏览"关系。

## Implementation Decisions

### 布局重新划分（4 区域）

```
┌────────────────────┐
│  Logo + 折叠按钮    │  ← Header
├────────────────────┤
│  [+ 新建申报单]     │  ← Primary CTA（白色圆角卡片，阴影，全宽）
├────────────────────┤
│  [搜索框]           │  ← Search
├────────────────────┤
│  申报单列表         │  ← List（flex-1，可滚动）
│  · 进口报关  待确认  │     每项：名称 + 类型·状态副标题
│  · 出口报关  已完成  │     左侧状态圆点，hover 灰色背景
├────────────────────┤
│  当前编辑: INV-001  │  ← Footer（固定底部，上下文信息）
└────────────────────┘
```

### CTA 按钮设计

- 白色背景 + 浅灰边框 + 微阴影（`shadow-sm`）
- 左侧圆形加号图标，右侧"新建申报单"粗体文字
- 全宽（减去 padding），圆角 10px
- hover 态边框变深，轻微上浮

### 列表项增强

- 主标题：申报单名称（bold，13px）
- 副标题：类型名称 + 状态文字 + 时间（11px，muted）
  - 格式：`{类型} · {状态} · {时间}`
  - 无类型时显示 `{状态} · {时间}`
- 右侧操作按钮（删除）保持 group hover 显示
- 选中态：浅灰色背景 + 左侧 2px 主题色竖线

### 底部信息栏

- 编辑模式：显示"当前正在编辑: {名称}" + 退出按钮
- 非编辑模式：显示"共 N 份申报单" + 上次更新时间
- 固定底部，上边框分割

### 空状态升级

- 居中图标（使用现有 IconDocument）
- 标题："准备开始"
- 描述："点击上方「新建申报单」创建第一份申报单"
- 淡化色调（opacity 降低）

## Testing Decisions

- 测试列表渲染（mock 数据验证主标题、副标题格式）
- 测试空状态显示
- 不需要：视觉回归测试（纯 UI 变更）

## Out of Scope

- 侧边栏宽度拖拽调整
- 列表拖拽排序
- 文件夹/分组功能
- 右键上下文菜单
- 列表项多选
