# Plan: v1.5 稳定打磨

> Source: `docs/02-roadmap.md` v1.5

## Phase 1.5a: 拖拽修复

### What to build

Electron 拖拽导入文件功能，从桌面直接拖文件到拖拽区即可导入。

### Acceptance criteria

- [ ] 从桌面/Finder 拖拽 PDF/DOCX/XLSX 文件到拖拽区，文件被识别并导入
- [ ] 拖拽 ZIP 压缩包，自动解压并导入内容文件
- [ ] 拖拽多个文件同时处理
- [ ] 拖拽时不支持的文件类型给出提示（而非静默忽略）
- [ ] `webUtils.getPathForFile` 调用正常，DevTools 控制台无相关错误

---

## Phase 1.5b: AI 提取稳定性

### What to build

AI 提取流程的边界情况处理：空文件、提取失败、超时等场景。

### Acceptance criteria

- [ ] 无导入文件时点提取，按钮禁用且 tooltip 提示"请先导入单证文件"
- [ ] AI API 超时/报错时，显示明确错误信息（非白屏或静默失败）
- [ ] 提取成功后 Block ②③ 正常展开，数据显示完整
- [ ] 连续提取两次，第二次的结果覆盖第一次（不叠加）

---

## Phase 1.5c: 存储验证

### What to build

确认 `~/Documents/DeclarAI/` 文件存储结构在实际使用中正确。

### Acceptance criteria

- [ ] 新建申报单后，对应文件夹创建（如 `D000001-(未命名)/`）
- [ ] 导入文件后，文件副本出现在 `files/` 子目录
- [ ] AI 提取后 `declaration.json` 包含完整数据
- [ ] 人工编辑保存后，`declaration.json` 数据更新
- [ ] 删除申报单后，对应文件夹被删除
- [ ] 旧版 userData 存储被自动清理

---

## Phase 1.5d: 类型切换稳定性

### What to build

7 种报关单类型之间来回切换，数据不丢失，UI 不崩溃。

### Acceptance criteria

- [ ] 通用视图 ↔ 各种类型切换，已编辑字段不丢失
- [ ] 切换类型时货物明细表列动态变化，数据保持
- [ ] 类型切换后保存，重新加载时类型状态恢复正确
- [ ] 选择类型后 declaration_outputs 表记录正确

---
