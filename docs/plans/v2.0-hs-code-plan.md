# Plan: v2.0 HS 编码 — 独立功能 + 内嵌联动

> Source: `v2.0-hs-code-prd.md` | 复用: hscoder 项目代码

## 架构定位

HS 归类是一个**可独立使用、也可内嵌使用**的功能模块。同一个后端引擎，两个前端入口。


## 两个入口

### 入口 A：独立「HS 归类咨询」（新增）

**位置**：侧边栏底部，新建按钮上方

**触发**：点击 → 主工作区切换为 HS 归类界面

**界面**：三块面板

```
┌──────────────────────────────────────────┐
│  🧠 HS 归类咨询                          │
├──────────────────────────────────────────┤
│                                          │
│  商品描述：                               │
│  ┌──────────────────────────────────────┐│
│  │ 真空泵，不锈钢材质，-30psi真空度，     ││
│  │ 1L/min抽气速率，134.4W功率，医疗用     ││
│  └──────────────────────────────────────┘│
│                              [开始归类]   │
│                                          │
│  ── 归类结果 ──                          │
│  ┌──────────────────────────────────────┐│
│  │ 📋 HS 编码: 8414.10.00.00            ││
│  │    货品名称: 真空泵                   ││
│  │    置信度: ● 高                       ││
│  │                                      ││
│  │ 关税税率: MFN 8%                      ││
│  │ 增值税: 13%                           ││
│  │ 监管条件: 无特殊监管                   ││
│  │                                      ││
│  │ 归类依据:                             ││
│  │ 根据归类总规则一，真空泵属于...         ││
│  │                                      ││
│  │ 候选编码:                             ││
│  │ · 8413.xx 液体泵 — 排除: 本品为气体泵  ││
│  │ · 8414.80 其他泵 — 排除: 有专用子目    ││
│  └──────────────────────────────────────┘│
│                                          │
│                    [保存为归类报告] [复制]  │
└──────────────────────────────────────────┘
```

- 输入商品描述（支持自然语言，类似 hscoder 的对话式）
- AI 返回完整归类分析
- 结果可复制，可保存为 markdown 报告到 `~/Documents/DeclarAI/HS归类/`
- 不关联任何申报单，独立使用

### 入口 B：货物明细内嵌（与 v2.0 PRD 一致）

**位置**：Block ③ 货物明细表的 HS 编码列

**触发**：点击货物行的 `[🔍 HS归类]` 按钮

**界面**：在行下方展开一个小面板，显示推荐编码 + 依据。用户点击确认填入。

**与入口 A 的关系**：入口 B 把货物名称自动填入入口 A 的引擎，但返回结果更简洁（只显示编码+置信度）。想深究的点"查看完整报告"跳转到入口 A。

## 代码复用

从 hscoder 直接移植以下代码（仅调整路径引用）：

| hscoder 源文件 | DeclarAI 目标 | 复用内容 |
|---------------|--------------|---------|
| `web/src/lib/skill-service.ts` | `src/main/hs-code/classifier.ts` | `extractKeywords()`、`searchTariff()`、`buildSystemPrompt()` |
| `ref/customs-hs-classifier/SKILL.md` | `prompts/hs-classifier-skill.md` | 完整归类规则（精简到 20KB 以内） |
| `ref/customs-hs-classifier/references/china-hscode-rule-2026.md` | `prompts/china-hscode-rule-2026.md` | 税则原文（3MB，直接复制） |

关键词词典是纯 TypeScript 数组，直接复制后调整品类/材质/功能三组词即可。

## IPC 通道

```
hs:classify     → 独立归类请求（商品描述文本 → 归类报告）
hs:classifyRow  → 内嵌归类请求（货物名称 + 上下文 → HS编码推荐）
hs:saveReport   → 保存归类报告到文件
```

## 实施顺序

| 步骤 | 内容 |
|------|------|
| 1 | 复制税则文件到 `prompts/` |
| 2 | 创建 `src/main/hs-code/` 模块（移植 hscoder 代码） |
| 3 | 注册 IPC 通道 |
| 4 | 实现入口 A：侧边栏按钮 + 独立归类界面 |
| 5 | 实现入口 B：货物明细 HS 列 + 行内推荐 |
| 6 | 报告保存功能 |

## 与现有申报单的关系

- 入口 A 归类结果**不关联申报单**，完全独立
- 入口 B 归类结果**写入对应货物行的 `hs_code` 字段**，随申报单保存
- 入口 A 的归类报告可以手动"应用到某票申报单"（后续功能）
