# PRD: HS 批量归类

> 日期: 2026-06-04 | 依赖: v2.0 HS 单条归类引擎

## 问题

报关员处理国际工程项目、汽车零部件进口等场景时，Excel 清单常有几十上百行品名需要逐一归类。单条模式逐行输入不现实。

## 解决方案

上传 Excel → AI 提取关键词 → grep 税则 → AI 一次性归类全部品名 → 结果表格 → 可复制/导出

## 流程

```
Excel 文件
  ↓ SheetJS 读取全部文本
  ↓
[LLM 第1轮] 从全部文本中提取 5-10 个税则检索关键词
  ↓
grep 搜索税则（所有品名共享一份搜索结果）
  ↓
[LLM 第2轮] 传入全部品名+规格+税则结果，一次性返回 JSON 数组
  ↓
解析展示表格 + 可导出 Excel
```

## LLM 响应格式

第 2 轮 LLM 调用一次性返回所有品名的归类结果：

```json
{
  "classifications": [
    {
      "row_index": 0,
      "product_info": "真空泵 / 不锈钢 / -30psi / 医疗用途",
      "hs_code": "8414.10.00.00",
      "hs_description": "真空泵",
      "confidence": "high",
      "mfn_rate": "8%",
      "vat_rate": "13%",
      "supervision_conditions": "无",
      "rationale": "根据归类总规则一，真空泵属第84章...",
      "tariff_text": "8414.10 真空泵...",
      "assumptions": null
    },
    {
      "row_index": 2,
      "product_info": "移动电源",
      "hs_code": "8507.60.00.90",
      "confidence": "low",
      "assumptions": "假设为锂离子蓄电池，容量约10000mAh，通用消费用途"
    }
  ]
}
```

置信度规则：
- `high`：用户提供的信息足够，不需 AI 假设，直接来自已知数据
- `low`：信息不足，AI 补充了假设。`assumptions` 字段必须说明具体假设了什么
- 批量模式默认跳过信息检查，AI 自行判断是否需假设

## 表格展示

| # | 商品信息 | HS编码 | 置信度 | 关税 | 增值税 | 监管 | 依据摘要 |
|---|---------|--------|--------|------|--------|------|---------|
| 1 | 真空泵 / 不锈钢... | 8414.10.00.00 | ● 高 | 8% | 13% | 无 | 第84章... |
| 2 | 移动电源 | 8507.60.00.90 | ● 低 | — | — | — | 基于假设 |

- 点击行展开：完整归类依据 + 税则原文 + AI 假设内容
- 低置信度行橙色背景
- 表格上方统计栏：总行数 / 高置信度 N / 低置信度 N
- 顶部操作：[复制表格] [导出 Excel]

## 导出 Excel

预置模板 .xlsx，使用 SheetJS 生成：

| 列 | 宽度 | 样式 |
|------|------|------|
| 序号 | 6 | 居中 |
| 商品信息 | 30 | 左对齐，自动换行 |
| HS编码 | 18 | Consolas 12px 紫色加粗 |
| 货品名称 | 22 | 左对齐 |
| 置信度 | 10 | 高=紫色底纹 / 低=橙色底纹 |
| 最惠国关税 | 10 | 右对齐 |
| 增值税 | 8 | 右对齐 |
| 监管条件 | 10 | 居中 |
| 归类依据摘要 | 35 | 左对齐，灰色 10px |
| AI 假设 | 30 | 橙色 10px（仅低置信度行） |

表头：深色背景 (#1E293B) 白色加粗 12px。奇数行浅灰背景 (#F8FAFC)。

## 前端入口

HS 预归类首页底部，「最近归类」上方：

> 批量归类：上传 Excel 清单，一次性处理多个品名 →

点击进入独立的批量归类页面（三态：空状态 / 处理中 / 结果表格）。

## 后端模块

- `src/main/hs-code/batch-classifier.ts`
  - `parseExcel(filePath)` → 返回全部文本（SheetJS 读取）
  - `batchClassify(excelText)` → 两轮 LLM + grep → 返回分类数组
- `src/main/ipc/hs-code.ts` 新增 `hs:batchClassify` handler
- preload 新增 `hsBatchClassify`
