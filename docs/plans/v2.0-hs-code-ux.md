### 模式切换 Tab

侧边栏 Header 下方，两个等宽的 Tab：

```
┌──────────────┐
│  📋 申报制单  │  ← active: 紫色背景 + 白色文字
├──────────────┤
│  🔍 HS 归类  │  ← inactive: 白色背景 + 灰色文字
└──────────────┘
```

- 点击切换 → 侧边栏内容变化 + 工作区内容变化
- 切换时保留各自的滚动位置和状态
- 默认打开上次关闭时的模式

## 数据存储

### DB 表

```sql
CREATE TABLE hs_classifications (
  id TEXT PRIMARY KEY,
  product_description TEXT NOT NULL,
  hs_code TEXT,
  hs_description TEXT,
  confidence TEXT,
  mfn_rate TEXT,
  vat_rate TEXT,
  supervision_conditions TEXT,
  full_result_json TEXT NOT NULL,    -- complete AI response
  report_path TEXT,                  -- saved report file path
  created_at TEXT NOT NULL DEFAULT (datetime('now','localtime'))
);
```

### 报告文件

保存路径：`~/Documents/DeclarAI/HS归类/[商品名]-[HS编码]-归类报告.md`

格式参考 hscoder 的归类报告模板。

## 与申报单的联动

在货物明细表中点击 `[🔍 HS归类]` 时：
1. 自动切换到 HS 归类模式
2. 货物名称预填入输入框
3. 归类结果出来后，用户可选择「应用到申报单」→ HS 编码写入该货物行

## UX 规则遵循

| 规则 | 应用 |
|------|------|
| `nav-state-active` | 模式 Tab 高亮当前模式（紫色背景） |
| `state-preservation` | 切换模式保留各自的滚动位置和输入状态 |
| `primary-action` | 每屏只有一个主按钮（制单=提取，归类=开始分析） |
| `empty-states` | 空状态有引导性说明 + 快捷示例 |
| `loading-states` | 分析中显示步骤进度而非纯 spinner |
| `progressive-disclosure` | 归类依据默认折叠，点展开看完整分析 |
| `multi-step-progress` | 分析过程显示 grep→AI 两步进度 |
