# PRD: DeclarAI HS 编码智能推荐

> 日期: 2026-06-04 | 参考: hscoder 项目架构

## 问题

DeclarAI 提取了商品名称（如"汽车配件"、"真空泵"），但没有 HS 编码。报关员需要离开软件，自己去查关税税则来确定编码。这是报关员每天耗时最多的工作。

## 解决方案

借鉴 hscoder 的成熟方案：**grep 关键词检索 + AI 推理归类**，不引入向量数据库。

```
货物名称 → 提取关键词 → grep 搜索本地税则文件 → 匹配行注入 prompt → AI 推荐 HS 编码 → 报关员确认
```

## 用户体验

### 快速模式（内置在提取流程中）

AI 提取完成后，每个货物明细行旁自动显示推荐的 HS 编码（4 位/6 位建议），附带置信度。报关员可点击展开查看归类依据。

### 专家模式（手动触发）

选中某行货物 → 点击「HS 归类分析」→ 系统搜索税则 → AI 给出完整归类报告（含多个候选编码、法律依据、税率、监管条件）。报告可保存到 `exports/`。

## 技术方案

### 税则数据

- 来源：`china-hscode-rule-2026.md`（3MB，19 万行，2026 版关税税则）
- 存储：打包在 `prompts/` 目录，随应用分发
- 更新：替换文件即可，无需改代码

### 关键词提取

预定义三组词典（来自 hscoder 已验证的分类体系）：
- 材质词：塑料、PP、钢、铁、铝、棉、丝、玻璃、陶瓷...
- 品类词：泵、阀、传感器、衬衫、零件、配件、机器、设备...
- 功能词：医疗、工业、农业、家用、商用、食品、汽车...

从货物名称中匹配关键词 → 用 grep 搜索税则文件 → 返回前 30 条匹配行号+内容。

### Prompt 构建

```
SYSTEM:
  # 海关商品归类专家
  （SKILL.md 核心归类规则 + 归类总规则 1-6）
  
  ## 从税则检索到的相关内容
  （grep 搜索结果，格式：行号:税则内容）
  
  ## 商品信息
  商品名称: {cargo_name}
  规格型号: {specification}
  用途: {从上下文中推测}
  
  请推荐最可能的 HS 编码（6位/10位），附带：
  - 官方货品名称
  - 归类依据
  - 候选编码及排除理由
  - 置信度（高/中/低）
```

### 输出解析

AI 返回结构化 JSON → 解析为 `HsCodeRecommendation` → 存入货物明细行：

```typescript
{
  cargo_name: "真空泵",
  hs_code_recommendation: {
    code: "8414.10.00.00",
    description: "真空泵",
    confidence: "high",
    rationale: "根据归类总规则一...",
    alternatives: [{ code: "8413.xx", reason: "排除: 本品为真空泵非液体泵" }],
    tariff_rate: "8%",
    vat_rate: "13%",
    supervision_conditions: "无特殊监管"
  }
}
```

### UI 变化

Block ③ 货物明细表中：
- 新增「HS 编码」列（如果选中类型的 cargo_column_mappings 包含 `hs_code`）
- 每行货物旁有「🔍 HS」按钮
- 点击后弹出 HS 推荐面板：显示推荐编码、置信度、归类依据
- 用户可接受推荐（填入编码）或拒绝（保留空白）

## 不做

- 不做完整的 HS 编码数据库查询界面
- 不做向量检索/RAG/嵌入
- 不做税率实时更新
- 不做监管证件的自动申请
- 不在提取阶段强制要求 HS 归类（可选步骤）

## 验收标准

- [ ] 输入"真空泵"，grep 能搜到税则中 8414 相关内容
- [ ] AI 返回的 HS 编码推荐包含编码、描述、置信度、依据
- [ ] 快速模式响应 < 10 秒（含 grep + AI 调用）
- [ ] 专家模式生成完整归类报告（可保存到 exports/）
- [ ] 用户接受推荐后，HS 编码写入货物明细的 `hs_code` 字段
- [ ] 税则文件可替换更新
