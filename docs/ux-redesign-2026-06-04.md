# DeclarAI UX 重构设计 — 三叠式分段流程

> 日期: 2026-06-04 | 基于 ui-ux-pro-max skill 研究

## 问题诊断

当前页面将"上传文件、AI提取、审核反馈、表单编辑、保存导出"全部堆在一个垂直滚动区域内，用户认知负担重，流程感缺失。

## 设计原则

- **Progressive Disclosure**: 逐步展开信息，先概览后细节
- **Progress Indicator**: 三块卡片天然形成进度感知
- **Content First**: AI 提取结果作为独立报告，而非静默填充
- **Single Primary Action**: 每块只有一个主要操作按钮

## 新布局：三叠式（Accordion Triptych）

```
┌──────────────────────────────────────────────────┐
│  转关运输货物申报单                                 │
│  预录入编号 · 状态                                  │
├──────────────────────────────────────────────────┤
│                                                  │
│  ┌─ ① 导入单证 ──────────────────────────────┐    │
│  │  拖拽区域 + 文件标签 + [AI 提取并审核]       │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  ┌─ ② AI 提取结果 ────────────────────────────┐  │
│  │  提取摘要 + 置信度 + 待确认问题列表           │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  ┌─ ③ 申报单表单 ── [保存] [导出] ──────────┐    │
│  │  运输信息表单 + 货物明细表格（内联标注）      │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
└──────────────────────────────────────────────────┘
```

## 区块说明

### ① 导入单证（始终可见）
- 拖拽区 + 文件标签列表
- 右上角 [AI 提取并审核] 按钮
- 提取中显示进度文案"AI 提取+审核中..."
- 最紧凑状态，无多余信息

### ② AI 提取结果（提取完成后展开）
- 展开动画 300ms ease-out
- 顶部摘要行：`✅ 提取完成 · 从 N 个文件中提取 · 整体置信度 高/中/低`
- 运输信息 KPI：已提取 X/5 字段
- 货物明细 KPI：已提取 N 行货物，共 X 件 Y KG
- 待确认问题列表（可折叠，>0 时默认展开）：
  - 每条问题 = 严重度色标 + 字段路径 + 问题描述
  - 0 个问题 → 显示"✅ 未发现明显问题，数据质量良好"
- 底部"跳转到表单 ↓"锚点按钮

### ③ 申报单表单（提取完成后展开）
- 运输信息表单（当前已有）+ 内联问题标注
- 货物明细表格（当前已有）+ 单元格 ⚠ 标注
- 顶部操作栏：[保存草稿 ⌘S] [导出 Excel]
- 编辑时自动标脏，Ctrl+S 保存

## 交互流

```
初始态: ① 展开 | ②③ 隐藏
    ↓ 用户导入文件 + 点击"AI 提取并审核"
提取中: ① 按钮 loading | ② 显示骨架屏
    ↓ 提取完成
完成态: ① 收起为紧凑模式 | ② 展开显示结果 | ③ 展开显示表单
    ↓ 用户确认问题 → 编辑表单 → 保存
```

## 视觉规范

- 三块卡片独立，块间距 24px
- 每块圆角 rounded-2xl (16px)，白色背景，shadow-card
- 区块标题 18px/600
- 展开动画：max-height transition 300ms ease-out
- 禁用按钮使用 primary-300（品牌化灰色）

## 技术实现要点

- 不需要路由，三块在同一页面用条件渲染 + CSS 动画控制显示
- 区块 ② 的数据来源于 AI 提取 API 返回的 extraction_notes + issues
- 区块 ③ 保留当前的 inline issue display 逻辑
- 操作按钮从页面 header 移到区块 ③ header
