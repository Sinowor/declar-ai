# DeclarAI 产品演进路线图

> 更新: 2026-06-06

## 产品哲学

1. **一个版本只解决一个核心问题**。不给用户堆功能。
2. **每个版本有独立的用户价值**。
3. **模块可组合**。用哪个开哪个。

---

## v2.5 费率计算器（新模块：calculator）

**问题**：报关员需手动查税率→计算关税+增值税+消费税→估算成本。效率低。

**核心功能**：
- 输入 HS 编码 + 原产国 + 货值 + 数量 → 一键出综合税费
- 关税 = 货值 × 税率（从税则文件解析）
- 增值税 = (货值 + 关税) × 增值税率
- 消费税 = (货值 + 关税) / (1 - 消费税率) × 消费税率（如适用）
- 结果卡片：分项明细 + 合计 + 完税总价
- 历史计算记录保存

**界面**：左侧输入 → 右侧结果卡片。可从归类模块跳转带入 HS 编码。

**数据源**：已有 `china-hscode-rule-2026.md` 税则文件，税率可从中解析。

## v2.6 知识库（新模块：knowledge）

**问题**：归类经验、客户特殊要求、口岸注意事项无法沉淀复用。

**核心功能**：
- 条目：标题 + HS 编码（可选关联）+ 标签 + Markdown 正文
- 标签：归类经验 / 客户备注 / 口岸须知 / 操作流程 / 自定义
- 搜索 + 标签过滤
- 与归类模块集成：HS 编码匹配时展示相关知识条目

**存储**：SQLite 本地表，Markdown 存储正文。

**界面**：左侧标签过滤 + 列表，右侧正文渲染。简洁备忘录风格。

## v3.0 制单内化：合规检查 + 审核工作流

两个功能都内化到现有的制单 AI 审核环节（Block ②）。

### 3.1 合规检查

在 Review Issues 中增加「合规风险」类别：

| 检查项 | 说明 |
|---|---|
| 许可证要求 | 该 HS 编码是否需进口/出口许可证 |
| 禁限商品 | 是否属于禁止/限制进出口目录 |
| 原产地证 | 是否适用优惠税率需提供原产地证 |
| 反倾销 | 原产国是否涉及反倾销税 |
| 检疫要求 | 是否需要动植物检疫/食品检验 |

高风险项红色提醒，需人工确认。数据来自税则文件中的监管条件字段。

### 3.2 审核工作流

状态流扩展：

```
草稿 → AI提取中 → 待审核 → 已审核 → 已完成
                    ↑         │
                    └─── 驳回 ──┘
```

- 审核人确认（姓名 + 日期）
- AI 提取值 vs 人工修改值字段高亮对比
- 审核意见备注 + 驳回原因记录

### v7.0 RAG 智能知识库（远期规划）

**核心能力**：服务端定时抓取海关政策/法规/公告 → AI 自动分块、向量化、入库。用户通过桌面端对话提问，服务端检索 + DeepSeek 生成回答，引用来源。用户也可上传自有文档到私有知识库。

#### 系统架构

```
┌─── 服务端 (轻量云服务器 2C4G) ────────────────────┐
│                                                    │
│  ┌─ Crawler ────────────────────────────────┐      │
│  │ 定时抓取 (每6h):                            │      │
│  │  customs.gov.cn — 海关总署公告               │      │
│  │  mofcom.gov.cn — 商务部贸易救济             │      │
│  │  关税司 — 税率调整通知                      │      │
│  │  RCEP/AEO/检疫 — 协定与认证                  │      │
│  └──────────────────────────────────────────┘      │
│                                                    │
│  ┌─ Pipeline ──────────────────────────────┐      │
│  │ 原文 → 清洗 → 分块(512 tokens) → embedding │      │
│  │ 去重: URL+内容相似度 >90% 跳过             │      │
│  │ 时效标签: 有效/失效/待确认                  │      │

└────────────────────────────────────────────────────┘
         │                          │

         │                          │
    ┌────┴────┐              ┌────┴────┐
    │ 用户A    │              │ 用户B    │
    │ DeclarAI │              │ DeclarAI │
    └─────────┘              └─────────┘
```

#### 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 后端运行时 | Bun / Node.js | 与前端同语言，Electron 生态复用 |
| 抓取 | `cheerio` + `node-cron` | 轻量 HTML 解析 + 定时任务 |
| 文本分块 | 手写 `RecursiveCharacterTextSplitter` | 200 行，不需要 LangChain |
| Embedding | `@xenova/transformers` + `all-MiniLM-L6-v2` | 384维，CPU 推理，npm 包即装 |
| 向量存储 | `hnswlib-node` 或 `sqlite-vec` | 纯内存/轻量持久化 |
| RAG 回答 | 现有 DeepSeek API | 复用，上下文 prompt 注入 |
| API 框架 | Express / Fastify | 成熟稳定 |
| 认证 | API Key + License | 与现有 License 系统对接 |

#### 知识库数据分层

| 层级 | 内容 | 来源 | 可见范围 |
|---|---|---|---|
| **公共层** | 海关公告、税率调整、检疫新规、RCEP 协定 | 服务端自动抓取 | 所有用户 |
| **私有层** | 用户上传的单证、笔记、客户要求 | 用户手动拖入 | 用户本人 |

#### 回答格式规范

每次回答必须：
- 引用具体来源（公告编号、发布日期）
- 标注信息时效性（有效/已废止/待确认）
- 如信息不足，明确声明「未找到相关规定」
- 区分「政策原文」和「AI 解读」

#### 部署与运维

- **服务器**：阿里云/腾讯云轻量 2C4G，月费 ~50 元
- **域名**：`kb.declarai.com`（可选）
- **爬虫频率**：每 6 小时，避开凌晨
- **存储**：文档原文 + 向量索引，初始 2GB，年增长 ~5GB
- **监控**：爬取失败告警、API 响应时间

#### 安全

- 用户私有文档加密存储，API Key 隔离
- 用户上传的文档不注入公共库
- API 请求限流（60次/分钟/用户）

#### 远期扩展

- 支持更多监管来源（FDA, EU, 检疫）
- 政策变更主动推送通知
- 多语言知识库（中/英/日/韩）
- 知识图谱（HS编码 ↔ 监管条件 ↔ 政策法规）

## 不做

单一窗口 API 直连、完整税则数据库、在线支付、多用户协作、移动端 App、物流追踪、财务结算。
