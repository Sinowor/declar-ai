# PRD: 附件管理模块

## Problem Statement

当前申报单工作流中，用户上传的源文件（箱单、发票、合同等）和 AI 生成的输出文件（提取结果、报关单等）缺乏统一管理界面。用户无法直观查看文件来源、类型标签，也无法快速打开或定位文件。所有文件散落在声明单的 `files/` 目录中，没有分类和检索能力。

## Solution

在制单工作区新增「附件管理」面板，统一展示当前申报单的所有关联文件，分为两大类别：

1. **上传文件**：用户导入的单证源文件。AI 提取时自动识别文件类型并打标签（如"箱单"、"发票"），用户可修改标签。支持系统默认程序打开 + 文件管理器定位两个操作。
2. **生成文件**：AI 和系统生成的输出产物（提取结果、各类报关单等）。展示文件名、用途说明、生成时间，同样支持打开和定位。

## User Stories

1. 作为制单员，我希望在工作区看到当前申报单的所有关联文件，以便一目了然地了解文件全貌。
2. 作为制单员，我希望上传的单证文件被 AI 自动识别并标注类型标签（箱单/发票/合同等），以便快速区分文件用途。
3. 作为制单员，我希望能够修改 AI 自动打上的文件标签，以便纠正 AI 可能的误判。
4. 作为制单员，我希望一个文件可以有 1~2 个标签（如一份文件同时是"箱单+发票"），以便准确描述复核用途的文件。
5. 作为制单员，我希望点击「打开」按钮能用系统默认程序直接打开文件，以便快速查看内容。
6. 作为制单员，我希望点击「定位」按钮能在文件管理器中显示文件位置，以便在文件夹中查找或复制文件。
7. 作为制单员，我希望看到每个生成文件的用途说明（如"进口报关单 - 正式申报版"），以便理解文件的使用场景。
8. 作为制单员，我希望看到文件的生成/上传时间，以便追溯文件时间线。
9. 作为制单员，我希望从历史申报单进入时也能看到当时的附件列表，以便回顾和审计。
10. 作为开发者，我希望附件元数据有稳定的数据模型，以便未来扩展更多文件类型和操作。

## 文件标签（File Tags）

### 上传文件标签

AI 在提取时自动识别文件内容类型并分配标签。标签为自由文本，但 AI 应优先从以下列表中选择：

| 中文标签 | 英文标识 | 说明 |
|---|---|---|
| 箱单 | packing_list | 装箱单 |
| 发票 | invoice | 商业发票 |
| 合同 | contract | 贸易合同 |
| 提单 | bill_of_lading | 海运/空运提单 |
| 运单 | waybill | 铁路/公路运单 |
| 原产地证 | certificate_of_origin | 原产地证明 |
| 报关单 | customs_declaration | 报关单草稿/底稿 |
| 其他 | other | 未能识别的文件 |

单文件支持 1~2 个标签（如一份 PDF 同时包含箱单和发票信息）。

### 标签编辑

- 点击标签弹出下拉菜单，从预设列表选择或直接输入自定义标签
- 取消标签（×按钮）删除该标签
- 最少保留 1 个标签
- 修改后的标签持久化到数据库

## 文件操作

### 打开（系统默认程序）

- 图标：📄 风格 SVG（文档+箭头）
- 行为：调用 `shell.openPath(filePath)` 以系统关联程序打开
- 不支持的文件类型显示灰色禁用态

### 定位（文件管理器）

- 图标：📁 风格 SVG（文件夹+箭头）
- 行为：调用 `shell.showItemInFolder(filePath)` 在 Finder/Explorer 中高亮文件
- 如果文件不存在则显示灰色禁用态并提示

## Implementation Decisions

### 数据模型

**`declaration_files` 表新增列：**

- `category` TEXT NOT NULL DEFAULT 'uploaded' — 文件类别：`uploaded`（上传文件）或 `generated`（生成文件）
- `tags` TEXT — JSON 数组字符串，如 `["箱单","发票"]`
- `purpose` TEXT — 用途说明（仅 generated 文件有效，uploaded 文件为 NULL）
- `output_type` TEXT — 输出文件类型键，如 `import_declaration`、`extraction_json`（仅 generated 文件有效）

兼容存量数据：`category` 默认 `uploaded`，`tags` 默认为 `["其他"]`，`purpose` 和 `output_type` 默认为 NULL。

### 数据库迁移

- 使用 `ALTER TABLE ADD COLUMN` 添加新列
- 存量数据自动补默认值
- 迁移脚本在 `initSchema()` 中执行，与现有表创建逻辑同位置

### AI 标签自动识别

- 在 `runAIExtraction()` 流程中增加标签识别步骤
- AI 分析 `file_name` + `extracted_text` 前 500 字，返回 1~2 个标签
- 识别失败或低置信度时默认为 `["其他"]`
- 标签在提取完成后立即写入数据库

### 前端组件

**新组件：`AttachmentPanel`**

- 位置：Workspace 中「③ 申报单数据」和「货物明细」之间，或作为独立的可折叠面板
- 布局：卡片式，与现有 Block 风格一致（`bg-white border rounded-2xl shadow-card`）
- 两个 Tab 或两个分段：上传文件 / 生成文件
- 每个文件行：图标 | 文件名 | 标签（可编辑）| 时间 | 操作按钮（打开 / 定位）
- 空状态：上传文件空 → "暂未导入文件"，生成文件空 → "暂无生成文件"

### IPC 通道

- `file:update-tags` — 更新文件标签
- `file:open` — 用系统默认程序打开
- `file:reveal` — 在文件管理器中定位
- `file:list-all` — 返回上传文件 + 生成文件的合并列表
- `file:add-generated` — 记录生成文件（系统内部调用）

### 标签编辑交互

- 点击标签文字 → 弹出 Popover，包含预设标签列表 + 自定义输入框
- 选择预设标签：点击即添加（替换当前选中的标签）
- 自定义标签：输入文本后回车确认
- 每个文件最多 2 个标签，达到上限时不再显示添加入口

## Testing Decisions

- 测试什么：数据库迁移的向前兼容性、标签的 CRUD 操作、文件打开/定位的 IPC 调用
- 不测试什么：AI 标签识别准确率（属于模型行为，非代码逻辑）、文件系统操作的具体结果（依赖 OS 环境）
- 测试风格：单元测试覆盖数据库操作（迁移、标签读写），集成测试覆盖 IPC 处理器

## Out of Scope

- 附件在线预览（PDF/图片内嵌查看器）
- 文件版本管理
- 文件拖拽排序
- 文件批量下载/打包
- 文件分享/发送功能
- 生成文件的自定义命名模板
- 跨申报单的文件引用/复制
- 文件加密存储

## Further Notes

- 此功能为制单工作流的补充体验，不改变现有的文件导入和 AI 提取流程
- 标签功能依赖 AI 模型对文件内容的分析能力，标签质量随模型能力提升
- `exports/` 目录已存在于存储结构中（`src/main/storage/index.ts:39`），生成文件存放在此目录
