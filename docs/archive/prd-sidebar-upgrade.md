# PRD: 侧边栏列表项信息增强

> 参考 WorkBuddy macOS 风格侧边栏设计，为列表项增加类型副标题

## Problem Statement

当前侧边栏申报单列表项信息密度低：每项仅显示名称、货物数、状态标签、时间。用户如果不点击进入，无法从列表中直接获知申报单类型（进口/出口/转关等），需要通过状态标签颜色推断部分信息。

## Solution

在列表项中增加申报类型副标题行，格式为 `{类型名称} · {状态文字} · {时间}`，让用户在不点击的情况下快速了解每份申报单的概况。

## User Stories

1. 作为制单员，我希望列表每项展示申报类型和状态文字，以便在不点击的情况下快速判断申报单类型和当前进度。
2. 作为制单员，我希望副标题格式统一清晰（如"转关运输 · 待确认 · 2小时前"），以便快速浏览。
3. 作为制单员，我希望未选择类型的申报单不显示类型字段（如"待确认 · 3天前"），以便区分已分类和未分类的申报单。

## Implementation Decisions

### 列表项布局（增强后）

```
┌────────────────────────────┐
│ ● INV-2024-001      🗑    │  ← 主标题 + 右侧操作
│   进口报关 · 待确认 · 2h前  │  ← 新增副标题行
└────────────────────────────┘
```

### 副标题规则

- 格式：`{类型名称} · {状态文字} · {相对时间}`
- 类型名称来自 `DeclarationTypeConfig.title`（如"进口报关"、"转关运输货物申报单"）
- 未选择类型时省略类型字段：`{状态文字} · {相对时间}`
- 字号 11px，颜色 `text-muted`
- 已有状态标签 badge 保持不变（彩色圆角标签）

### 类型名称映射

通过 `typeConfigs` 查找 `type` 对应的 `title`。类型 config 在 `DeclarationTypeConfig.title` 中定义。如果 `type` 为 null 或未找到对应 config，不显示类型。

## Testing Decisions

- 测试副标题格式化逻辑（有类型/无类型/类型名过长截断）
- 验证现有列表项其他字段（货物数、状态标签、时间）不受影响

## Out of Scope

- 侧边栏布局重构
- CTA 按钮位置调整
- 空状态设计变更
- 底部信息栏新增
