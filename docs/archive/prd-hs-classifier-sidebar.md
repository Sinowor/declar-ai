# PRD: HS 归类模块 — 历史记录侧边栏

## Problem Statement

当前 HS 归类模块的归类历史通过两种方式展示：（1）空状态页面底部显示最近 5 条记录，（2）点击「查看全部」后跳转到全屏历史页面，完全离开分类功能。这两种方式都割裂了用户的归类工作流——查看历史意味着离开当前归类上下文。

## Solution

为 HS 归类模块新增常驻历史侧边栏，设计与制单模块的 Sidebar 完全一致：

- 左侧历史列表，右侧分类主界面
- 支持折叠（0 宽度），展开按钮在 NavRail
- 搜索过滤历史记录
- 点击历史项直接回填到分类输入框并展示结果
- 与制单 Sidebar 共享折叠状态和 NavRail 展开按钮

## User Stories

1. 作为归类员，我希望在分类页面左侧看到归类历史列表，以便在不离开当前页面的情况下浏览和复用历史记录。
2. 作为归类员，我希望侧边栏可以折叠，以便扩大分类主界面的工作空间。
3. 作为归类员，我希望折叠后的侧边栏完全消失（0 宽度），展开按钮出现在 NavRail 中，以便与制单模块保持一致的交互体验。
4. 作为归类员，我希望在历史列表中搜索商品名称或 HS 编码，以便快速找到某次归类记录。
5. 作为归类员，我希望点击历史记录项后，该商品的描述和 HS 编码自动回填到分类界面，以便快速复核或修改。
6. 作为归类员，我希望侧边栏的视觉风格与制单 Sidebar 完全一致（卡片式、深色模式适配、列表间距），以便获得统一的产品体验。

## Implementation Decisions

### NavRail 折叠按钮复用

- 当前 NavRail 已有一个「侧栏」展开按钮，通过 `sidebarCollapsed` 控制显示
- HS 模块和制单模块不会同时展开——用户一次只在一个模块
- 但它们的折叠状态是独立的（`hsSidebarCollapsed` 和 `sidebarCollapsed`）
- NavRail 需要接收两个模块的折叠状态，任一折叠时显示展开按钮

### 修改 `App.tsx` 中的 NavRail Props

```typescript
<NavRail
  sidebarCollapsed={activeModule === 'declarations' ? sidebarCollapsed : hsSidebarCollapsed}
  onToggleSidebar={() => {
    if (activeModule === 'declarations') setSidebarCollapsed(v => !v)
    else setHsSidebarCollapsed(v => !v)
  }}
/>
```

### 数据源

- 使用现有 IPC `hsHistory`（已在 preload 中暴露）
- 搜索结果缓存于 state，搜索为客户端过滤

### 文件改动

| 文件 | 改动 |

| `HsClassifier.tsx` | 外层 flex 容器包裹 Sidebar + 主界面 |
| `App.tsx` | 新增 `hsSidebarCollapsed` 状态，传 NavRail |
